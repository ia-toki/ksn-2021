Description

Pak Dengklek has $Q$ mischievous ducks, numbered from $1$ to $Q$. Pak Dengklek gives a number to each of his ducks, i.e. duck $i$ gets a number $P_i$. However because Pak Dengklek's ducks are mischievous, they hide their own number using a binary cipher with key $K$. The rule of the binary cipher for a key $K$ is changing a number $P_i$ into $C_i$ that is bitwise OR of $P_i$ and $K$.

Now Pak Dengklek only knows the values of $C_i$ for each of his ducks. Fortunately, Pak Dengklek knows the used cipher key $K$. Pak Dengklek asks for your help to restore the value of $P_i$ for each of his ducks. If there are more than one possible numbers, you may output any possible number. If there is no possible number, you have to tell Pak Dengklek by outputting $-1$.

What is bitwise OR?
Bitwise OR of two integers $x$ and $y$ can be computed as follows.
  1. Write $x$ and $y$ in base two with the same bit length.
  2. Perform the $\text{OR}$ operation for each corresponding bit pair:
    • $0 \text{ OR } 0 = 0$
    • $0 \text{ OR } 1 = 1$
    • $1 \text{ OR } 0 = 1$
    • $1 \text{ OR } 1 = 1$
  3. Convert the $\text{OR}$ result back to base ten.
As an example, $6 \text{ OR } 10 = 14$. Because $6 = 0110$ and $10 = 1010$ in base two, the bitwise OR obtained is $1110 = 14$ in base ten.

Constraints

  • $1 \le Q \le 100\;000$
  • $0 \le K, C_i \le 2^{30} - 1$

Subtasks

  1.   (9 points) Contains only the following test case:
    $Q = 5$, $K = 2$, and the values of $C_i$ are $1$, $2$, $3$, $4$, and $5$ respectively.
  2. (11 points) $K = 0$
  3. (12 points) $C_i \lt K$
  4. (21 points) $Q \le 1000$, $K, C_i \le 2^{10} - 1$
  5. (19 points) $K, C_i \le 2^{20} - 1$
  6. (28 points) No additional constraints.

Explanation of Sample

For duck $2$, there is no possible value of $P_2$ so you have to output $-1$.

For duck $3$, the possible values of $P_3$ are $8$, $9$, $12$, and $13$. You may output any possible number.
